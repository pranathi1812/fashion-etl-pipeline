# Fashion Retail ETL Pipeline & Analytics Dashboard

An end-to-end data engineering project that extracts raw fashion retail sales data, transforms and cleans it using Python and Pandas, loads it into a MySQL database, and visualizes insights through an interactive web dashboard — automated via AWS Lambda.

## Project Overview

This project simulates a real-world data pipeline used by fashion retail businesses to track sales performance, customer behavior, and product trends. Built entirely from scratch as a portfolio project to demonstrate data engineering and analytics skills.

## Architecture

Kaggle CSV Dataset → Extract (Python/Pandas) → Transform (Clean, Standardize, Remove Nulls) → Load (MySQL Database) → Export (data.json) → Dashboard (HTML/CSS/JavaScript/Chart.js) → AWS Lambda (Automated Daily Scheduling)

## Dashboard Features

- Total Revenue KPI — overall sales performance

## ETL Pipeline Steps

### 1. Extract
- Reads raw CSV dataset (3,400 rows) using Pandas
- Loads into Python DataFrame for processing

### 2. Transform
- Standardizes column names (lowercase, underscores)
- Removes duplicate records
- Converts data types (text to numbers)
- Removes rows with missing purchase amounts
- Result: 2,750 clean records ready for analysis

### 3. Load
- Connects to MySQL using mysql-connector-python
- Creates fashion_sales table if not exists
- Truncates old data and inserts fresh records
- Commits 2,750 rows to database

### 4. Export
- Calculates summary statistics using Pandas
- Exports KPIs and chart data as dashboard/data.json
- Dashboard reads this file to render visualizations

## SQL Analysis (10 Queries)

- Query 1: Business overview — revenue, records, averages
- Query 2: Top 10 selling items by purchase count
- Query 3: Revenue and transactions by payment method
- Query 4: Monthly sales trend over time
- Query 5: Customer spending segmentation (Budget to Luxury)
- Query 6: Top rated items (HAVING, WHERE, IS NOT NULL)
- Query 7: Payment method preference by spending segment
- Query 8: Item revenue ranking using RANK() window function
- Query 9: Month-over-month growth using LAG() and CTE
- Query 10: Full customer summary with VIP/Regular/Occasional tiers

## 🚀 How to Run Locally

1. Clone the repository
git clone https://github.com/YOUR_USERNAME/fashion-etl-pipeline.git

2. Install dependencies
pip install pandas mysql-connector-python numpy

3. Download dataset from Kaggle
Search "Fashion Retail Sales Dataset" and save as data/fashion_retail_sales.csv

4. Set up MySQL
CREATE DATABASE fashion_db;

5. Add your MySQL password to DB_CONFIG in etl_pipeline.py

6. Run the pipeline
python etl_pipeline.py

7. Open dashboard/index.html in your browser

## AWS Lambda Deployment

The pipeline runs automatically on AWS Lambda triggered by EventBridge on a daily cron schedule (0 6 * * ? *) — refreshing MySQL with fresh data every day automatically.
